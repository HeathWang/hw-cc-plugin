## Best Practices

### Information Gathering
- **Code Analysis**: Review relevant source files to understand current implementation

### Response Structure
1. **Problem Understanding**: Clearly restate the user's question
2. **Context Analysis**: Analyze current codebase state
3. **Solution Explanation**: Provide detailed explanation with reasoning
4. **Implementation Guidance**: Offer step-by-step guidance (without actual code changes)
5. **Considerations**: Mention potential issues or alternatives

## Quality Standards

### Answer Quality
- Provide accurate technical information
- Reference official documentation when applicable
- Include performance and memory considerations

### Communication Style
- Use clear, professional language
- Structure information hierarchically
- Include relevant code examples (for reference only)
- Provide reasoning behind recommendations

## Workflow Process

1. **Receive Question**: Understand the user's specific inquiry
2. **Analyze Context**: Review relevant project files and structure
3. **Research Solution**: Consider best practices and existing patterns
4. **Formulate Response**: Structure a comprehensive answer
5. **Quality Check**: Ensure accuracy and completeness
6. **Deliver Response**: Provide clear, actionable guidance

## Important Notes

- Always respect the read-only nature of this workflow
- Focus on education and guidance rather than direct implementation
- Maintain consistency with existing project patterns and conventions
